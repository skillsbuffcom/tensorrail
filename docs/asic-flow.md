# TensorRail ASIC Flow

TensorRail is primarily an FPGA accelerator proof-of-concept, but the repo also
contains an experimental OpenLane flow for generating real IC physical-design
artifacts from the compute core.

## What Is Real Evidence?

Real ASIC evidence is generated by tools from source files:

```text
RTL
  -> OpenLane / OpenROAD
  -> synthesized netlist
  -> placed and routed DEF
  -> final GDS/OAS
  -> KLayout/Magic DRC
  -> Netgen LVS
  -> OpenSTA timing
  -> power report
```

The conceptual SVG at `layout/tensorrail_floorplan.svg` is useful for explaining
the architecture, but it is not signoff evidence. It should be labeled as a
floorplan preview unless a CI run has produced GDS/OAS and reports.

## OpenLane Target

The ASIC flow uses a small wrapper around the MAC array:

- top module: `tensorrail_asic_top`
- source files:
  - `rtl/mac_cell.v`
  - `rtl/systolic_array.v`
  - `rtl/tensorrail_asic_top.v`
- OpenLane config: `asic/openlane/tensorrail_mac_tile/config.json`
- pin order: `asic/openlane/tensorrail_mac_tile/pin_order.cfg`
- PDK: `sky130A`
- standard-cell library: `sky130_fd_sc_hd`
- clock: `clk`
- initial clock period: `20 ns`

The FPGA board top in `rtl/top.v` is intentionally not the ASIC top. It includes
board-level UART and PSRAM stubs. The ASIC wrapper exposes the compute macro
interface that OpenLane can synthesize, place, and route cleanly.

## Run Command

From the repo root:

```bash
cd asic/openlane/tensorrail_mac_tile
python3 -m openlane --pdk-root "$PDK_ROOT" --run-tag syqnal_ci config.json
```

Expected outputs after a successful run:

```text
asic/openlane/tensorrail_mac_tile/runs/syqnal_ci*/results/final/gds/*.gds
asic/openlane/tensorrail_mac_tile/runs/syqnal_ci*/results/final/def/*.def
asic/openlane/tensorrail_mac_tile/runs/syqnal_ci*/reports/**/*
```

## Syqnal Display Contract

Syqnal should display:

- `Floorplan Preview` for hand-authored SVG diagrams
- `GDS/OAS Layout Preview` only for previews rendered from real GDS/OAS
- `DRC Clean` only from Magic/KLayout report evidence
- `LVS Matched` only from Netgen report evidence
- `Timing Closed` only when STA reports non-negative WNS and zero failing endpoints
- power numbers only from OpenROAD/OpenLane reports

All evidence should be tied to:

- commit SHA
- runner version
- tool name/version
- artifact path

## Generated Artifact Policy

Do not commit complete OpenLane `runs/` directories. They are too large and too
noisy for source control.

Commit source and configuration:

- RTL
- OpenLane config
- pin constraints
- documentation

Use CI artifacts for generated outputs:

- GDS/OAS
- DEF
- reports
- logs
- rendered previews
